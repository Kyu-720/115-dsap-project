# 文字搜尋系統

## Proposal Report

### 動機與目標
<!-- 說明為什麼想做這個專題 -->
動機：傳統搜尋方式，像是瀏覽器內建搜尋：Ctrl + F，在面對複雜查詢條件或不完整輸入時，查詢表達能力有限。因此希望實作一個文字搜尋系統，使用不同的搜尋演算法，使使用者即使只記得部分內容，或進行條件式查詢，也能獲得較符合需求的搜尋結果。

目標：建立一個以文字資料為主的搜尋系統，用Boolean Search與Prefix Search，結合關鍵字排序，使搜尋結果有更大的彈性與相關性。

### 預期功能
<!-- 列出預計實作的功能 -->
1. 關鍵字搜尋
2. 搜尋方法：
- Boolean Search(布林搜尋)
- Prefix tree(前綴詞搜尋)
3. 關鍵字相關性與排序

### 使用技術
<!-- 使用的語言、框架、工具等 -->
開發環境：Jupyter Notebook  
程式語言：Python  
搜尋對象：以英文文字資料為主  
相似搜尋系統分析：
1. 瀏覽器內建搜尋：Ctrl + F  
    (1)運行環境：網頁瀏覽器  
    (2)搜尋對象：頁面的文字內容  
    (3)功能:  
    - 單一關鍵字搜尋
    - 有逐筆跳轉功能，可快速瀏覽搜尋結果
  
    (4)限制:  
    - 沒有多關鍵字查詢
    - 無Boolean Search
    - 無搜尋結果排序
    - 無法搜尋多個文件

2. Chrome 擴充工具：Multiple Search and Highlight  
    (1)運行環境：Google Chrome 瀏覽器  
    (2)搜尋對象：頁面的文字內容  
    (3)功能:   
    - 多關鍵字搜尋
    - 有顏色標記，可快速辨識不同關鍵字
    - 顯示各關鍵字出現的次數  
    
    (4)限制:  
    - 多關鍵字只能同時搜尋，沒有Boolean Search
    - 無搜尋結果排序
    - 無法搜尋多個文件

### 與課程的關聯
<!-- 你的專題可能涉及哪些資料結構或演算法概念？為什麼？ -->
1. 資料結構 
- Trie tree(字典樹) : 快速搜尋拼字不完整的單字。
- Hash(雜湊表) : 做inverted index（倒排索引），加快搜尋時間
2. 演算法
- Boolean Search(布林搜尋) : 用集合運算：& / |，實作 AND / OR
- Ranking(排序) : 對結果進行排序，讓較相關的文件排序在前
